# Термометр-сигнализатор на ESP32

**Платформа:** Wemos TTGO ESP-WROOM-32 с OLED 0.96"
**Датчик:** DS18B20 (GPIO27)
**Зуммер:** пассивный (GPIO25)
**Кнопка:** Boot (GPIO0, встроенная)
**Дисплей:** SSD1306 128×64, I2C — SDA=GPIO5, SCL=GPIO4

---

## Необходимые библиотеки

Установить через **Arduino IDE → Manage Libraries**:

## WiFi и MQTT

### Настройки подключения

Хранятся в **arduino_secrets.h** (не в репозитории):

```cpp
#define WIFI_SSID1     "Keenetic-1693"     // первичная сеть
#define WIFI_PASS1     "..."
#define WIFI_SSID2     "Keenetic-115"      // резервная сеть
#define WIFI_PASS2     "..."

#define MQTT_HOST      "192.168.100.119"
#define MQTT_PORT      1883
#define MQTT_CLIENT_ID "esp32-thermometer"
#define MQTT_USER      "myuser"
#define MQTT_PASS      "..."
```

### MQTT-топики

| Топик | Содержимое | Retained |
|---|---|---|
| `thermometer/temperature` | Температура, напр. `24.3` | ✅ |
| `thermometer/status` | `OK` или `ALARM` | ✅ |
| `thermometer/base` | Базовая температура | ✅ |

### Логика подключения при старте

1. Пробуем **Keenetic-1693** (до 8 сек)
2. Если не найдена — пробуем **Keenetic-115** (до 8 сек)
3. Если WiFi OK, пробуем MQTT (до 4 сек, с авторизацией)
4. Если сеть недоступна → экран ошибки 2 сек → автономный режим

### Когда отправляются данные

| Событие | Отправка |
|---|---|
| Вход в рабочий режим | ✅ однократно |
| Плановая | ✅ раз в час |
| Тревога активна | ✅ при каждом чтении датчика (раз в 5 сек) |
| Снятие тревоги | ✅ однократно |
| Повторная попытка | через 60 сек после неудачи |

### Автономный режим

Если WiFi или MQTT недоступны — устройство работает полностью автономно:
- Отображение температуры, пороги, зуммер — всё работает
- На экране в строке статуса отображается **AUTO** вместо **WiFi**
- Повторные попытки подключения каждые 60 сек при следующей отправке

## Энергосбережение

| Параметр | Значение | Описание |
|---|---|---|
| CPU частота | 80 МГц | снижена с 240 МГц по умолчанию |
| WiFi TX power | 11 dBm | снижена для экономии энергии |
| Light sleep | 50 мс | между итерациями loop, когда WiFi выключен |
| Brownout detector | отключен | предотвращает случайные перезагрузки |
| WiFi отключается | автоматически | когда тревога не активна |

## Логика работы

### 1. Режим стабилизации

Устройство наблюдает за температурой в течение **3 минут**.
Если за это время показания не выходят за пределы **±0.1°C** — температура фиксируется как **базовая (Base)**.

Если отклонение больше ±0.1°C — трёхминутный отсчёт начинается заново.

---

### 2. Рабочий режим

**При входе в рабочий режим верхний порог не задан** — зуммер молчит.
Первое нажатие кнопки устанавливает верхний порог (+0.2°C) и активирует контроль.

#### Пороги срабатывания

| Порог | Значение | Регулируется |
|---|---|---|
| Верхний (Hi) | Base + delta | ✅ Кнопкой |
| Нижний (Low) | Base − 0.3°C | ❌ Фиксированный |

---

### 3. Режим калибровки

Вход: **длинное нажатие** (≥ 800 мс).
Выход: **длинное нажатие** — поправка сохраняется, → стабилизация.

Диапазон: **от −4.0°C до +4.0°C**, шаг 0.1°C.
Сохраняется в EEPROM.

---

## Управление кнопкой Boot

| Действие | Рабочий режим | Калибровка | Стабилизация |
|---|---|---|---|
| **Короткое нажатие** | Регулировка Hi | Поправка +0.1°C | Принудит. рабочий |
| **Длинное нажатие** | → Калибровка | → Стабилизация | → Калибровка |

## Индикация на экране

### Рабочий режим
```
       24.6 C           ← температура (крупно)
─────────────────────
Hi +0.5 (24.8 C)        ← верхний порог
─────────────────────
НОРМА              WiFi  ← статус + сеть
```

Если WiFi выключен: `НОРМА  AUTO`
При тревоге: `!! ТРЕВОГА !!`

### Отправка данных
```
   Sending data...
     Please wait
```
Показывается на время WiFi/MQTT передачи (до ~20 сек в худшем случае).

### Стабилизация
```
   Стабилизация...
       24.6 C
   Осталось: 120 сек
   [████░░░░░░░░░░░░]
```

### Ошибка сети
```
    Сеть недоступна
    Автономный режим
```

## Звуковые сигналы

| Событие | Сигнал |
|---|---|
| Включение | 1 длинный писк |
| База зафиксирована | 2 коротких писка |
| Нажатие кнопки (короткое) | 1 тихий писк |
| Нажатие кнопки (длинное) | 1 низкий писк |
| Тревога | 3 писка каждые 3 секунды |

---

## Подключение

```
ESP32 GPIO27  →  DS18B20 DATA  (+резистор 4.7 кОм между DATA и VCC)
ESP32 GPIO25  →  Зуммер +
ESP32 GND     →  Зуммер −
ESP32 3.3V    →  DS18B20 VCC
ESP32 GND     →  DS18B20 GND
```

## Настройки в скетче

```cpp
#define WIFI_CONNECT_TIMEOUT_MS   8000UL   // ожидание WiFi (мс)
#define MQTT_CONNECT_TIMEOUT_MS   4000UL   // ожидание MQTT (мс)
#define WIFI_RETRY_INTERVAL_MS   60000UL   // интервал повторных попыток (мс)
#define MQTT_PUBLISH_INTERVAL_MS (60*60*1000UL)  // плановая отправка (1 час)
#define READ_INTERVAL_MS          5000UL   // интервал опроса датчика (мс)
#define STAB_WINDOW_MS  (3UL*60UL*1000UL) // время стабилизации
#define ALARM_LOW_DELTA           0.3f     // нижний порог от базы
#define ALARM_HIGH_START          0.2f     // начальный верхний порог
#define ALARM_HIGH_MAX            1.0f     // максимальный верхний порог
#define CPU_FREQ_MHZ              80       // частота CPU (МГц)
#define SLEEP_US                50000UL    // light sleep (мкс)
```
