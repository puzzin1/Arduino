# Термометр-сигнализатор на ESP32

**Платформа:** Wemos TTGO ESP-WROOM-32 с OLED 0.96"  
**Датчик:** DS18B20 (GPIO27)  
**Зуммер:** пассивный (GPIO25)  
**Кнопка:** Boot (GPIO0, встроенная)  
**Дисплей:** SSD1306 128×64, I2C — SDA=GPIO5, SCL=GPIO4

---

## Необходимые библиотеки

Установить через **Arduino IDE → Manage Libraries**:

| Библиотека | Автор |
|---|---|
| ESP8266 and ESP32 OLED driver for SSD1306 | ThingPulse |
| OneWire | Paul Stoffregen |
| DallasTemperature | Miles Burton |

## Логика работы

### 1. Режим стабилизации (сразу после включения)

Устройство наблюдает за температурой в течение **3 минут**.  
Если за это время показания не выходят за пределы **±0.1°C** от первого замера — температура признаётся стабильной и фиксируется как **базовая (Base)**.

Если в любой момент температура отклоняется больше ±0.1°C — трёхминутный отсчёт **начинается заново**.

На экране отображаются:
- текущая температура (крупно)
- обратный отсчёт в секундах
- прогресс-бар

---

### 2. Рабочий режим

После фиксации базы устройство переходит в рабочий режим.

**При входе в рабочий режим верхний порог не задан** — зуммер молчит.  
Первое нажатие кнопки устанавливает верхний порог и активирует контроль.

#### Пороги срабатывания зуммера

| Порог | Значение | Регулируется |
|---|---|---|
| Верхний (Hi) | Base + delta | ✅ Кнопкой |
| Нижний (Low) | Base − 0.3°C | ❌ Фиксированный |

Зуммер срабатывает если:
- температура **поднялась** выше верхнего порога, **или**
- температура **упала** ниже нижнего порога (Base − 0.3°C)

При тревоге зуммер пищит **3 раза каждые 3 секунды**, пока температура не вернётся в норму.

#### Что показывает экран в рабочем режиме

```
       24.6 C           ← текущая температура (крупно)
─────────────────────
Base: 24.3 C
Hi:  +0.5  (24.8 C)    OK / !!
Low: -0.3  (24.0 C)
```

---

### 3. Режим калибровки

Вход: **длинное нажатие** кнопки (≥ 800 мс) из любого режима.  
Выход: **длинное нажатие** — поправка сохраняется, устройство переходит в стабилизацию заново.

Позволяет ввести постоянную поправку к показаниям датчика.  
Поправка применяется ко всем измерениям:

```
T_итого = T_датчик + поправка
```

Диапазон поправки: **от −4.0°C до +4.0°C**, шаг 0.1°C, по кругу.

Калибровка сохраняется в **EEPROM** и не теряется при отключении питания или перезагрузке.

На экране отображаются:
- текущая поправка (крупно, со знаком)
- сырое значение датчика
- итоговая температура с учётом поправки

## Управление кнопкой Boot

| Действие | Рабочий режим | Режим калибровки | Стабилизация |
|---|---|---|---|
| **Короткое нажатие** | Регулировка верхнего порога | Поправка +0.1°C | — |
| **Длинное нажатие** (≥ 800 мс) | Войти в калибровку | Выйти, сохранить | Войти в калибровку |

---

## Регулировка верхнего порога (кнопка в рабочем режиме)

| Нажатие | Порог (delta) |
|---|---|
| 1-е | +0.2°C |
| 2-е | +0.3°C |
| 3-е | +0.4°C |
| … | … |
| 9-е | +1.0°C |
| 10-е (по кругу) | +0.2°C |

## Звуковые сигналы

| Событие | Сигнал |
|---|---|
| Включение | 1 длинный писк |
| База зафиксирована | 2 коротких писка |
| Нажатие кнопки (короткое) | 1 короткий тихий писк |
| Нажатие кнопки (длинное) | 1 низкий писк |
| Тревога | 3 писка каждые 3 секунды |

---

## Сохранение калибровки в EEPROM

ESP32 имеет встроенную энергонезависимую память. Калибровочная поправка записывается в EEPROM при каждом изменении в режиме калибровки и при выходе из него. При включении устройство автоматически загружает сохранённое значение.

Для сброса калибровки к нулю: войдите в режим калибровки, нажимайте кнопку пока поправка не станет **0.0**, затем выйдите длинным нажатием.

---

## Подключение

```
ESP32 GPIO27  →  DS18B20 DATA  (+резистор 4.7 кОм между DATA и VCC)
ESP32 GPIO25  →  Зуммер +
ESP32 GND     →  Зуммер −
ESP32 3.3V    →  DS18B20 VCC
ESP32 GND     →  DS18B20 GND
```

Кнопка Boot и OLED уже встроены в плату, дополнительного подключения не требуют.

## Настройки в скетче

```cpp
#define STAB_WINDOW_MS   (3UL * 60UL * 1000UL)  // время стабилизации (мс)
#define STAB_TOLERANCE   0.1f   // допуск стабилизации (°C)
#define ALARM_LOW_DELTA  0.3f   // нижний порог от базы (°C)
#define ALARM_HIGH_START 0.2f   // начальный верхний порог (первое нажатие)
#define ALARM_HIGH_MAX   1.0f   // максимальный верхний порог
#define LONG_PRESS_MS    800    // порог длинного нажатия (мс)
```
